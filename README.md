## 5. Limitations 및 개선방향</p>
### Limitations
&#160; &#160; &#160; &#160; &#160; - 수요조사, 데이터 수집, 전처리, 알고리즘을 디테일하게 구현하는데 프로젝트 기간이 다소 제한적<br/>
#### 1) 모델 학습측면
&#160; &#160; &#160; &#160; &#160; - 문장에서 단어의 위치에 따라 개체명 인식 차이 발생<br/>
&#160; &#160; &#160; &#160; &#160; - 선호/비선호/알러지 표현으로 분류시 모호한 부분으로 인한 개체명 인식 차이 발생<br/>
&#160; &#160; &#160; &#160; &#160; => 다양한 개체명 조합을 통한 추가적인 데이터 증식 필요<br/>
&#160; &#160; &#160; &#160; &#160; => 모호한 표현들에 대한 개체명 구체화 필요
#### 2) 알고리즘 측면
&#160; &#160; &#160; &#160; &#160; - 음식 추천 가중치 (frequency)에 대한 명확하고 현실감 있는 로직 구현에 대한 아쉬움
#### 3) 코딩 측면
&#160; &#160; &#160; &#160; &#160; - 유저의 다양한 입력에 대해 대처할 수 있는 디테일한 코딩에 대한 아쉬움
